# Drawings → VOR Skill

## Цель

Превратить строительные чертежи в структурированную ВОР (Ведомость объёмов работ) — 
единственный источник данных для сметы. **Целевое качество: 80-85% соответствия 
профессиональной ВОР сметчика.**

---

## 🎯 Ключевое правило: ВОР ≠ Спецификация

**Спецификация РД** = материалы на 1 изделие.
**ВОР** = работы и материалы на ВЕСЬ объект с нормами расхода.

| Что | Спецификация РД | ВОР |
|-----|----------------|-----|
| Бетон | м³ на 1 плиту | м³ × кол-во плит × 1,015 (потери) |
| Плёнка | м² на 1 плиту | м² × кол-во × 1,22 (нахлёсты) |
| Арматура | кг по спецификации | кг из ведомости стали |
| Окраска | Нет | м² закладных × норма |

## 📋 Пайплайн: 5 фаз (обязательный порядок)

```
[Чертежи PDF] → [Фаза 1: Чек-лист листов] → [Фаза 2: Общие указания] 
                                                  ↓
[ВОР Excel] ← [Фаза 5: Валидация] ← [Фаза 4: Извлечение] ← [Фаза 3: Анализ]
                  ↑
            OCR: image-ocr / Tesseract / PaddleOCR
```

**Без прохождения всех 5 фаз — ВОР считается черновиком.**

## 🔍 OCR-стратегия: 4 уровня fallback

**Интеграция со skills:** `image-ocr` (если доступен), `drawing-ocr` (для сканов)

### Уровень 1: Текстовый слой PDF (предпочтительный)

```bash
# Проверить, есть ли текстовый слой
pdftotext input.pdf output.txt
# Если текст есть — использовать его, игнорировать OCR
```

**Когда:** PDF экспортирован из AutoCAD/Revit, не скан.
**Точность:** Высокая (100% если шрифт встроен).

### Уровень 2: Vision Model (основной для схем)

```
[PDF страница] → [PNG 300 DPI] → [Vision Model (Kimi/k2p6)] → [Подсчёт элементов]
```

**Когда:** Схемы расположения плит, узлы, разрезы.
**Точность:** Средняя-высокая (зависит от качества изображения).
**Ограничения:** Размер изображения < 25M пикселей.

### Уровень 3: OCR с предобработкой (таблицы, спецификации)

```python
# Стратегия для мелкого текста (таблицы спецификаций, ведомость стали)
1. Конвертировать страницу в PNG с 400+ DPI
2. Deskew (выпрямить поворот текста)
3. Увеличить контраст (CLAHE)
4. Бинаризация (Otsu)
5. OCR по ячейкам таблицы (не всей страницей)
6. Попробовать 2 движка: Tesseract + PaddleOCR
```

**Инструменты:**
- `image-ocr` skill — если доступен в системе (предпочтительный)
- `Tesseract` — fallback, `/usr/bin/tesseract` установлен
- `PaddleOCR` — лучше для русского, таблиц, мелкого шрифта
- `EasyOCR` — мультиязычность

**Когда:** Спецификации, ведомость стали, таблицы арматуры.

### Уровень 4: Делегирование пользователю (критичные данные)

Если OCR даёт < 70% точности на таблице арматуры/спецификации:

```
1. Выделить проблемную область на скриншоте
2. Отправить пользователю: "Не могу распознать таблицу арматуры на листе 5. 
   Пришли, пожалуйста, Excel/CSV с ведомостью расхода стали"
3. Получить данные → вставить в ВОР без потери точности
```

**Правило:** Лучше спросить и получить точные данные, чем догадываться.

## Поддерживаемые форматы

| Формат | Обработка | Точность | OCR |
|--------|-----------|----------|-----|
| PDF (с текстовым слоем) | Чтение текста | Высокая | Не нужен |
| PDF (скан/растр) | Vision model + OCR | Средняя-высокая | image-ocr / Tesseract |
| PNG/JPG | Vision model + OCR | Средняя-высокая | image-ocr / Tesseract |
| DWG | ❌ Не поддерживается | — | Конвертировать в PDF |

## Фаза 1: Чек-лист по листам (ОБЯЗАТЕЛЬНО)

Перед анализом — пройтись по каждому листу. Ни один лист не пропускать.

### Шаблон чек-листа:

```markdown
## Лист 1: Титул / Общие данные
- [ ] Ведомость рабочих чертежей (все ли листы присланы?)
- [ ] Общие указания (прочитать полностью — каждое предложение = потенциальная позиция)
- [ ] Нормативные ссылки (СП, ГОСТ — проверить актуальность)
- [ ] Условия площадки (грунты, грунтовые воды)

## Лист 2-N: Схемы / Планы
- [ ] Масштаб указан?
- [ ] Все оси пронумерованы?
- [ ] Плиты/элементы посчитаны?
- [ ] Закладные детали учтены?
- [ ] Деформационные швы отмечены?

## Лист N+1: Спецификации
- [ ] Все позиции перенесены в ВОР?
- [ ] Ведомость расхода стали извлечена?
- [ ] Материалы с нормами расхода?

## Лист N+2: Узлы / Разрезы
- [ ] Армирование узлов?
- [ ] Сложные конструкции (каналы, ниши)?
- [ ] Дополнительные закладные?
```

**Файл:** `references/checklist_sheets.md` (копировать для каждого проекта)

## Фаза 2: Общие указания → Позиции ВОР (КРИТИЧНО)

**Самая частая ошибка:** прочитать общие указания, но не превратить в позиции.

### Правило: Каждое требование = строка в ВОР

| Текст общих указаний | Позиция в ВОР | Код ГЭСН |
|---------------------|---------------|----------|
| "Противоусадочные швы через 6 м" | Нарезка швов, герметик | УТОЧНИТЬ |
| "Закладные защитить от коррозии" | Окраска закладных | ГЭСН46-08-022-03 |
| "Обратная засыпка ЩПС с уплотнением" | Засыпка, уплотнение | ГЭСН01-02-061 |
| "Ограждение котлована в ППР" | Временное ограждение | УТОЧНИТЬ |
| "Мониторинг геотехнический" | Геотехнический мониторинг | УТОЧНИТЬ |

### Чек-лист общих указаний:

**Файл:** `references/checklist_general_instructions.md`

```markdown
## Общие указания → ВОР (авто-проверка)
- [ ] Противоусадочные швы → нарезка + герметик
- [ ] Деформационные швы → ППС + герметик + трубы ПЭ
- [ ] Защита закладных → грунт + эмаль
- [ ] Обратная засыпка → ЩПС + уплотнение
- [ ] Ограждение котлована → щиты / ограждение
- [ ] Дренаж / отвод воды → дренажные работы
- [ ] Мониторинг → геодезия / контроль
- [ ] Бетонирование при t<0°С → утепление / добавки
- [ ] Консервация → защита конструкций
```

**Правило:** Если в общих указаниях есть глагол ("выполнить", "устроить", "защитить", 
"произвести") — это работа, и она должна быть в ВОР.

## Фаза 3: Анализ (Проход 1 — структура)

**Цель:** Понять структуру, не извлекать цифры.

### Что анализировать:
1. **Группировка элементов** — как профи группирует по маркам (П1→Пм1→Пм2→Пнм1)
2. **Типовые элементы** — сколько экземпляров каждого типа?
3. **Масштаб** — без масштаба нет объёмов
4. **Условные обозначения** — легенда, типы линий

---

## Фаза 4: Извлечение (Проход 2 — количества)

### Правило 1: Типовые элементы × Количество

**Если на чертеже:** «П1 — 89 шт»
**В ВОР:**
- НЕ «Плита П1 — 1 шт»
- ДА «Плита П1 — 89 шт» → бетон 173,1 × 89? Нет!

**Уточнение:** Спецификация даёт объём на ВСЕ плиты данного типа (173,1 м³ = сумма 
на все П1), а не на 1 шт. Проверить по ведомости!

### Правило 2: Нормы расхода (обязательны)

**Файл:** `knowledge_base.md` — раздел «Нормы расхода материалов»

| Материал | Норма расхода | Источник | Пример |
|----------|--------------|----------|--------|
| Бетон тяжёлый | ×1,015–1,02 | ГЭСН | 173,1 → 176,6 м³ |
| Плёнка полиэтиленовая | ×1,22 | ГЭСН | 979,4 → 1198,8 м² |
| Пенополистирол ППС15 | ×1,02 | ГЭСН | 11,1 → 11,3 м³ |
| Пенополистирол ППС25 | ×1,03 | ГЭСН | 57 → 58,7 м³ |
| Герметик | ×1,0 | — | без потерь |
| Уплотнитель Вилатерм | ×1,01–1,02 | ГЭСН | 3684,9 → 3751 м |
| Грунт ЭП-057 | ×1,1 | практика | 0,3 → 0,33 кг |
| Эмаль ХС 5132 | ×1,1 | практика | 0,6 → 0,66 кг |

**Формула:** `ВОР = Спецификация_РД × Норма_расхода`

### Правило 3: Закладные → Защита

Каждая закладная требует защиты от коррозии:

```
Площадь окраски = Σ (кол-во закладных × площадь поверхности 1 шт)
```

| Закладная | Масса 1 шт | Площадь окраски | Кол-во | Итого площадь |
|-----------|-----------|-----------------|--------|---------------|
| МН118-3 | 3,5 кг | ~0,1 м² | 10 | 1,0 м² |
| МН540 | 8,5 кг/м | ~0,2 м²/м | 11,2 м | 2,24 м² |
| МН3 | 4,9 кг | ~0,15 м² | 5 | 0,75 м² |

**Грунт:** площадь × 0,3 кг/м²
**Эмаль:** площадь × 0,6 кг/м² (2 слоя)

### Правило 4: Деформационные швы → комплект

Деф.шов = не только герметик, но и:
- Пенополистирол ППС15 (заполнитель)
- Труба ПЭ 80 SDR13,6 Ø25×2 (водоотвод/компенсация)
- Уплотнитель Вилатерм
- Полиуретановый герметик

### Правило 5: Арматура = Ведомость стали

**Ведомость расхода стали (лист 6) — эталон.**

Проверка: сумма масс по спецификации = ведомость?
- Если расхождение >2% — перепроверить

## Фаза 5: Валидация (4-глазый метод)

### Автоматическая проверка:

```bash
python3 scripts/validate_vor.py --vor ВОР.xlsx --drawing <pdf>
```

**Проверки:**
1. [ ] Все листы чертежа обработаны?
2. [ ] Все общие указания → позиции?
3. [ ] Нормы расхода применены?
4. [ ] Закладные → защита?
5. [ ] Деформационные швы → комплект?
6. [ ] Типовые элементы × количество?
7. [ ] Нет «УТОЧНИТЬ» без пометки?
8. [ ] Единицы измерения стандартные (ФЕР-2020)?

### Ручная проверка (3-5 позиций):

Сверить с чертежом:
- 1 позиция из спецификации
- 1 позиция из общих указаний  
- 1 позиция с масштабом
- 1 позиция с нормой расхода
- 1 позиция с типовым элементом

## 📐 Формат ВОР (профессиональный)

**Группировка:** По маркам элементов (как в файле профи)

```
Раздел 1. Устройство монолитной плиты пола на отм. 0,000
  └─ П1 (бетонная плита)
     ├─ Укладка плёнки
     ├─ Плёнка (с нормой)
     ├─ Бетон (с нормой)
     ├─ Деформационные швы
     └─ Материалы швов (с нормами)
  └─ Пм1 (железобетонная)
     ├─ Пенополистирол
     ├─ Бетон (с нормой)
     ├─ Закладные
     ├─ Окраска закладных
     ├─ Арматура
     └─ Деформационные швы
  └─ Пм2 (железобетонная)
     └─ ...
Раздел 2. Устройство пандуса
  └─ Пнм1
Раздел 3. Устройство противоусадочных швов
```

**Шаблон:** `templates/ВОР_шаблон_профи.xlsx`

| Колонка | Содержание | Пример |
|---------|-----------|--------|
| № п/п | Порядковый | 1, 2, 3... |
| Наименование | Полное, без сокращений | "Укладка полиэтиленовой плёнки 200 мкм" |
| Ед. изм. | Стандарт ФЕР | м², м³, кг, шт, м |
| Кол-во | С нормой расхода | 1198,786 |
| Примечание | Обязательно! | "с учётом нормы расхода" |

## 🚫 Критические ошибки (red lines)

| Ошибка | Последствие | Как избежать |
|--------|------------|--------------|
| Пропустить общие указания | -5–10 позиций | Чек-лист Фазы 2 |
| Без норм расхода | Занижение 15–25% | Таблица норм |
| Не учесть защиту закладных | Пропуск 2–4 позиций | Правило 3 |
| Не умножить типовые элементы | Занижение в N раз | Правило 1 |
| Пропустить деф.швы/трубы | Пропуск 3–5 позиций | Правило 4 |
| Спецификация = ВОР | Нет норм, нет работ | Понимать разницу |
