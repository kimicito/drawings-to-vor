# Drawings → VOR Skill

## Цель

Превратить строительные чертежи в структурированную ВОР (Ведомость объёмов работ) — единственный источник данных для сметы.

## OCR-стратегия: 3 уровня fallback (обновлено 2026-06-30)

**Проблема:** На проекте «Компрессорная станция» OCR провалился на таблицах арматуры (лист 5), спецификациях закладных (листы 5–7). Профи извлёк 20+ позиций арматуры, я — 0.

### Уровень 1: Текстовый слой PDF (предпочтительный)

```bash
# Проверить, есть ли текстовый слой
pdftotext input.pdf output.txt
# Если текст есть — использовать его, игнорировать OCR
```

**Когда:** PDF экспортирован из AutoCAD/Revit, не скан.
**Точность:** Высокая (100% если шрифт встроен).

### Уровень 2: OCR с предобработкой (основной)

```python
# Стратегия для мелкого текста (таблицы спецификаций)
1. Конвертировать страницу в PNG с 400+ DPI
2. Deskew (выпрямить поворот текста)
3. Увеличить контраст (CLAHE)
4. Бинаризация (Otsu)
5. OCR по ячейкам таблицы (не всей страницей)
6. Попробовать 2 движка: Tesseract + PaddleOCR
```

**Инструменты:**
- `PaddleOCR` — лучше для русского, таблиц, мелкого шрифта
- `Tesseract` — fallback, нужен fine-tune для технических терминов
- `EasyOCR` — мультиязычность

### Уровень 3: Делегирование пользователю (критичные данные)

Если OCR даёт < 70% точности на таблице арматуры/спецификации:

```
1. Выделить проблемную область на скриншоте
2. Отправить пользователю: "Не могу распознать таблицу арматуры на листе 5. 
   Пришли, пожалуйста, Excel/CSV с ведомостью расхода стали"
3. Получить данные → вставить в ВОР без потери точности
```

**Правило:** Лучше спросить и получить точные данные, чем догадываться.

## Поддерживаемые форматы

| Формат | Обработка | Точность |
|--------|-----------|----------|
| PDF (с текстовым слоем) | Чтение текста + визуальный анализ | Высокая |
| PDF (скан/растр) | Vision model (полностью) | Средняя-высокая |
| PNG/JPG | Vision model | Средняя-высокая |
| DWG | ❌ Не поддерживается. Конвертировать в PDF | — |

## Архитектура процесса

```
[Чертежи PDF/PNG] → [Prepare] → [Analyze] → [Extract] → [Validate] → [VOR Excel] → [Smeta]
                      ↓              ↓           ↓           ↓
                src/prepare    Vision +      Структура    src/validate
                _drawing.py    OCR         позиций      _vor.py
```

## Пайплайн (4-глазый метод)

### Фаза 1: Подготовка (Prepare)

**Проверить перед анализом:**

- [ ] **Масштаб указан?** Если нет — запросить у пользователя. Без масштаба невозможны объёмы.
- [ ] **Количество листов?** Все ли прислали? Проверить нумерацию.
- [ ] **Тип чертежей:** Планы, разрезы, фасады, схемы ВК, спецификации?
- [ ] **Качество:** Читаемы ли размеры, подписи, обозначения?

**Если PDF многостраничный:**
```bash
# Разбить на страницы (автоматически)
python3 src/prepare_drawing.py --input чертеж.pdf --output pages/
```

### Фаза 2: Анализ (Analyze) — Проход 1

**Цель:** Понять структуру чертежа, не извлекать цифры.

**Что смотреть:**
1. Заголовок — объект, этаж, дата, масштаб
2. Условные обозначения — легенда, типы линий
3. Основные элементы — стены, перекрытия, двери, окна, оборудование
4. Схемы инженерных систем — трассы, узлы учёта, венткамеры

**Выход:** Структурная карта чертежа (список элементов без количеств).

### Фаза 3: Извлечение (Extract) — Проход 2

**Цель:** Извлечь конкретные позиции с количествами и единицами измерения.

**По каждому элементу:**

| Поле | Правило | Пример |
|------|---------|--------|
| Наименование | Как в чертеже/спецификации, без сокращений | «Установка вентиляторов радиальных ВР 80-75 №6,3» |
| Ед. изм. | Стандартная для данного типа работ | шт, м², м³, пог.м, компл |
| Количество | Из чертежа (с масштабом) или спецификации | 1, 20, 45.2 |
| Примечание | Особенности, типы, размеры | «периметром до 1600 мм» |
| Шифр/марка | Обозначение на чертеже | ВР 80-75 №6,3 |

**Правила извлечения:**
- **Планы этажей:** Считать площади (размеры × масштаб)
- **Разрезы:** Считать высоты, объёмы
- **Спецификации оборудования:** Копировать как есть (количества уже указаны)
- **Схемы трубопроводов/воздуховодов:** Считать длины по линиям (масштаб)
- **Узлы:** Считать количество по плану (привязка к помещениям)

**Критичные проверки:**
- Если количество неочевидно — пометить «УТОЧНИТЬ»
- Если единица измерения нестандартная — проверить и записать стандартную
- Если позиция на чертеже есть, но в спецификации нет — записать, пометить «Проверить спецификацию»
- Если позиция в спецификации есть, но на чертеже нет — записать, пометить «Проверить чертёж»

### Фаза 4: Валидация (Validate)

**Двойная проверка (4 глаза):**

```bash
# Автоматическая проверка ВОР
python3 src/validate_vor.py --vor ВОР.xlsx --drawing_pages pages/
```

**Что проверяет:**
- Все ли позиции с чертежа попали в ВОР?
- Нет ли дублей?
- Корректны ли единицы измерения?
- Сходятся ли суммы (если есть промежуточные итоги)?
- Нет ли «пустых» строк?

**Ручная проверка (я):**
- Сверить 3-5 случайных позиций с чертежом
- Проверить, что масштаб применён правильно
- Проверить, что все типы работ охвачены

### Фаза 5: Передача в Smeta

```
ВОР.xlsx → data/<object_name>/ВОР.xlsx
           ↓
         make eval SMETA=... VOR=... (smeta pipeline)
```

## Типы чертежей и особенности

### Планировочные решения (АР)
- **Извлечь:** Площади помещений, периметры, высоты помещений
- **Ед. изм.:** м², м³, пог.м
- **Особенность:** Масштаб критичен для площадей

### Конструктивные решения (КР/КЖ)
- **Извлечь:** Объёмы бетона, арматуры, площади опалубки
- **Ед. изм.:** м³, т, м²
- **Особенность:** Толщины из разрезов

### Инженерные системы (ОВ/ВК/ЭО/СС)
- **Извлечь:** Длины трасс, количество оборудования, узлов
- **Ед. изм.:** пог.м, шт, компл
- **Особенность:** Схемы + спецификации оборудования вместе

### Спецификации (отдельные листы)
- **Извлечь:** Как есть, с проверкой соответствия чертежам
- **Ед. изм.:** Как указано
- **Особенность:** Могут быть неполными или устаревшими

## Критические правила (red lines)

1. **Нет масштаба = нет объёмов.** Запросить масштаб или отказаться от расчёта.
2. **ВОР — единственный источник.** Если позиция в ВОР, но не в смете — ошибка. Если в смете, но не в ВОР — ошибка.
3. **«УТОЧНИТЬ» лучше, чем наугад.** Любая неопределённость → пометка + вопрос пользователю.
4. **Спецификации и чертежи могут расходиться.** Фиксировать расхождения, не «исправлять» самостоятельно.
5. **Единицы измерения — стандартные.** ФЕР-2020 использует определённые единицы. Привести к ним.

## Чек-лист перед передачей ВОР в смету

- [ ] Масштаб проверен и указан
- [ ] Все листы обработаны
- [ ] Все позиции из спецификаций перенесены
- [ ] Все позиции с чертежей (узлы, оборудование) перенесены
- [ ] Дублей нет
- [ ] Единицы измерения стандартные
- [ ] 3-5 позиций сверены с чертежами (ручная проверка)
- [ ] validate_vor.py пройден без FAIL
- [ ] Пользователь подтвердил ВОР

## Интеграция с smeta

```bash
# После формирования ВОР
cd /path/to/smeta
make eval SMETA=data/<object>/смета_v1.xlsx VOR=data/<object>/ВОР.xlsx
```

## Формат ВОР (Excel)

Шаблон: `templates/ВОР_шаблон.xlsx`

| Колонка | Содержание | Обязательно |
|---------|------------|-------------|
| № п/п | Порядковый номер | Да |
| Наименование работ | Полное название | Да |
| Ед. изм. | Единица измерения | Да |
| Количество | Число | Да |
| Шифр/марка | Обозначение на чертеже | Нет |
| Примечание | Особенности | Нет |
| Источник | На каком листе/чертеже найдено | Рекомендуется |

### Вспомогательные работы (автоматически добавлять)

| Позиция | Когда добавлять | Ед. изм. | Примечание |
|---------|----------------|----------|------------|
| Временное ограждение | Всегда для стройплощадки | шт | Секции 2×3 м |
| Перевозка грунта | При выемке | т | Масса грунта × 1.65 |
| Работа на отвале | При выемке | 1000 м³ | Объём выемки |
| Щиты опалубки | Для каждого бетонного элемента | м² | По периметру × высота |
| Уплотнение ПГС | При устройстве ПГС | м³ | Объём × 1.18 × 1.01 |

### Пример: Фундаменты (типовые элементы)

**Если на чертеже:** «Фундамент Фом1 — 4 шт, объём 19.21 м³»

**В ВОР добавить:**
1. Бетон В25: 19.21 × 4 = 76.84 м³ (не 19.21 м³!)
2. Бетон В7.5 (подготовка): 2.67 × 4 = 10.68 м³
3. Щиты опалубки: 21.9 × 4 = 87.6 м²
4. Арматура (все позиции): × 4
5. Каркасы КП: × 4
6. Закладные: × 4

**Формула:** Объём_1шт × Количество = Итоговый_объём

### Пример расхождения (моя ВОР vs профи) — Компрессорная станция

| Позиция | Моя ВОР | Профи | Причина расхождения |
|---------|---------|-------|---------------------|
| Бетон плиты | 469 м³ | 476 м³ | Профи считает с запасом |
| Арматура плиты | 78 стержней | 60 т | Я не извлёк полную таблицу |
| Опалубка | — | 133 м² | Я не включил щиты |
| Закладные | 4 шт | 2.16 т | Я не извлёк все типы |
| Анкерные блоки | — | 0.89 т | Я не включил |
| Фундаменты Фом1 | 19.21 м³ | 77.99 м³ | Я не умножил на 4 шт |
| Крыльцо | — | 1.58 м³ | Я не извлёк полностью |

**Вывод:** Сметчик всегда умножает типовые элементы на количество, включает все детали из спецификаций, добавляет опалубку и вспомогательные работы.
