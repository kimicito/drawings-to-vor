# Drawings → VOR → Smeta Pipeline

## Архитектура пайплайна

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Чертежи PDF   │ ──▶ │  drawings-to-vor │ ──▶ │     smeta       │
│  (КЖ, КР, АР)   │     │  (ВОР.xlsx)     │     │  (Смета.xlsx)   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │                       │
        ▼                       ▼                       ▼
   PDF/PNG/JPG            Excel с объёмами        ЛСР с ценами
   Масштаб, спецификации  Позиции, ед.изм,        ФЕР-2020, БИМ
                          количества              НР, СП, ФССЦ
```

**Цель:** 80-85% соответствия профессиональной ВОР сметчика.

**Связь:** `drawings-to-vor` — это **Stage 1** пайплайна. Его выход (`ВОР.xlsx`) — это **единственный вход** для `smeta` (Stage 2).

**Eval — обязательный:** ВОР без eval = черновик. Передавать в smeta нельзя.

## Быстрый старт (полный пайплайн)

```bash
# === STAGE 1: Чертежи → ВОР ===
cd skills/drawings-to-vor

# 1. Подготовить чертежи (PDF → PNG, определение масштаба)
python3 src/prepare_drawing.py --input чертеж.pdf --output pages/

# 2. OCR-обработка (таблицы, спецификации)
# → Использовать image-ocr skill (если доступен)
# → Fallback: Tesseract / PaddleOCR
python3 src/ocr_pipeline.py --input pages/ --output ocr_results/

# 3. Создать шаблон ВОР (формат профи)
python3 src/create_vor_template.py --template templates/ВОР_шаблон_профи.xlsx

# 4. Заполнить ВОР (вручную или с LLM)
# → открыть templates/ВОР_шаблон_профи.xlsx, заполнить
# → Группировка по маркам элементов (П1→Пм1→Пм2→Пнм1)
# → Нормы расхода обязательны (см. knowledge_base.md)

# 5. Валидировать ВОР (8 проверок)
python3 scripts/validate_vor.py --vor ВОР.xlsx --drawing_pages pages/
# → Проверки: листы, общие указания, нормы, закладные, деф.швы, типовые элементы

# 6. Eval ВОР — сравнение с профессиональным эталоном
python3 scripts/eval_vor.py --vor ВОР.xlsx --example data/vor_structure_example.json
# → Если FAIL: исправить ВОР, перезапустить eval
# → Если PASS: можно передавать в smeta

# === STAGE 2: ВОР → Смета ===
cd ../smeta

# 7. Скопировать ВОР в data проекта
cp ../drawings-to-vor/ВОР.xlsx data/<object>/ВОР.xlsx

# 8. Составить смету (БИМ-методика, ФЕР-2020)
# → использовать SKILL.md smeta

# 9. Проверить смету
python3 eval_smeta.py --smeta смета.xlsx --vor data/<object>/ВОР.xlsx
```

## Ключевые правила (v2.0)

### 1. ВОР ≠ Спецификация
- Спецификация РД = материалы на 1 изделие
- ВОР = работы и материалы на ВЕСЬ объект с нормами расхода

### 2. 5 фаз (обязательно)
1. Чек-лист по листам — ни один лист не пропускать
2. Общие указания → позиции — каждый глагол = работа
3. Анализ — группировка по маркам (П1→Пм1→Пм2→Пнм1)
4. Извлечение — нормы расхода обязательны
5. Валидация — 8 проверок + ручная сверка 3-5 позиций

### 3. Нормы расхода (обязательны)
| Материал | Норма | Пример |
|----------|-------|--------|
| Бетон | ×1,015–1,02 | 173,1 → 176,6 м³ |
| Плёнка ПЭ | ×1,22 | 979,4 → 1198,8 м² |
| ППС15 | ×1,02 | 11,1 → 11,3 м³ |
| ППС25 | ×1,03 | 57 → 58,7 м³ |

### 4. OCR-стратегия
1. Текстовый слой PDF (предпочтительный)
2. Vision Model (для схем)
3. OCR с предобработкой (таблицы) — image-ocr / Tesseract / PaddleOCR
4. Делегирование пользователю (если < 70% точности)

## Интеграция со skills

| Skill | Роль | Когда использовать |
|-------|------|-------------------|
| `image-ocr` | OCR таблиц, спецификаций | Таблицы арматуры, ведомость стали |
| `drawing-ocr` | OCR сканов чертежей | Старые сканы, нет текстового слоя |
| `smeta` | Составление сметы | После PASS eval ВОР |
